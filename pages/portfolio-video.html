Cowork Render

The exported ZIP is a browser animation — not a video. Claude Cowork built the render pipeline and took four attempts to get it right.

1
24fps — zoom transitions looked choppy
Standard film rate. The easing curves needed 60fps to look smooth. Bumped.
2
60fps but duplicate frames — still wrong
React schedules state updates asynchronously. Screenshots were taken before React committed the new frame. Fixed by injecting a useEffect signal — captures only after React confirms the frame is fully rendered.
3
Frame sync fixed — added TTS audio, then removed it
Clean motion. Google TTS narration added. Listened to it once. Removed it. Generic voice doesn't add anything to a visual-first portfolio video.

Try it yourself

Want to build something similar? These are the exact prompts — adapted for your own project. Start with Claude Design, then bring the exported ZIP to Cowork.

Claude Design Opening prompt
Create a crisp animated portfolio video using my brand colors and design system. The audience is [e.g. recruiters / hiring managers] looking for [your role title]. Structure it as: hook (identity reveal) → what I do (3 disciplines) → key stats (numbers that matter) → 2–3 case studies with live demos → outro with CTA. Use typography-led animation — text driving the story, not decorating it. Reference my design system files for colors, fonts, and token values. QA every scene for layout issues before handing off.
Attach your design system file
Claude Cowork Render prompt
I have a Claude Design export (ZIP file) attached. It's a React animation that plays in the browser. Please render it as a proper MP4 video. Requirements: - 60fps, 1920×1080 - H.264, high quality (CRF 14) - No audio needed (or add: mix in [track name] CC music at low volume) - Trim to exact animation duration - Save to my Downloads folder Use Playwright headless Chrome to capture frames. Make sure React has fully committed each frame before screenshotting — don't use a fixed timeout, use a render signal.
Attach the exported ZIP
Tip 01
Import your design system. Use the Import button in Claude Design to attach your brand tokens file. The model reads it before writing a single line of animation code.
Tip 02
Push back early. The first build is a draft. Ask it to change anything that doesn't feel right — tone, content, layout. It's a conversation, not a one-shot prompt.
Tip 03
Cowork needs context. Tell it the target resolution, frame rate, and quality. Without these it'll pick defaults. Mention the render-signal requirement — it prevents duplicate frames.

Both tools are research previews. What took four render attempts today will likely take one in six months. The process will get faster. The direction is already clear.
