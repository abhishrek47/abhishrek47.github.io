How I Built This
✓ What's real in this demo
🔊Voice narration via Web Speech API — your browser speaks your name and role on click
⚙Role-adaptive content — narration, scenario question and feedback all change based on selected role
📋xAPI-style log structure — the sidebar fires real event names, verbs and data payloads matching actual xAPI spec
🔀Branching logic — correct/incorrect paths with consequence feedback, same pattern used in Storyline
~ What's simulated
🌐The Saba LMS API call — no real LMS is connected; learner name and role come from the form input
🎙Murf AI voice — the browser speech engine is a placeholder; the production version used Murf AI for natural human-quality narration
🔑Learner ID and prior completions — randomised on each session to illustrate the data structure
→ Production architecture
Storyline 360 ──OnEnter──▶ JS trigger │ └──▶ fetch() ──▶ Saba LMS REST API │ // returns learnerName, role, dept └──▶ Murf AI API ──▶ audio blob │ // POST { text, voiceId, style } └──▶ Audio element ──▶ plays in slide │ └──▶ SetVar() ──▶ Storyline adapts content
