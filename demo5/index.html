How I Built This
✓ Real & Accurate
28 hand-authored storyboards — 4 topics × 7 ID models, every screen written by an instructional designer (me), not generated randomly
Each ID model produces structurally distinct output — ADDIE phases, Bloom cognitive levels, Gagné's 9 Events, SAM iterations, Merrill's principles, 4C/ID task types, and Kirkpatrick's backward design are all genuinely different
Narration scripts, visual descriptions, interaction types, and design rationale notes are all real ID deliverables — written to the standard I would submit to a client
The ID Model Analysis panel (right) accurately maps each screen to its model phase — that mapping is genuine, not decorative
~ Simulated / Simplified
Topic selection is fixed to 4 pre-built scenarios — a production tool would accept any free-text topic and generate via API
A Cloudflare Worker proxy is already built and deployed — the switch to live Claude API generation requires only adding API credits
The ~1 second loading delay is intentional — it simulates real generation latency and signals that meaningful processing is happening
Export to DOCX or Google Slides is not wired up in this demo — the storyboard JSON is fully structured and ready for that pipeline
→ In Production
Live API generation via Cloudflare Worker: any topic, any audience, any ID model — storyboard built from scratch in 3–6 seconds
Export pipeline: storyboard JSON → DOCX (Articulate Storyline-ready) or Google Slides via API, preserving narration, visuals, and interaction specs
SME review workflow: comment threads per screen, tracked changes, approval status per slide — stored in SharePoint or LMS
Version history: every regeneration saved with timestamp and model used — compare ADDIE vs Bloom output side by side
