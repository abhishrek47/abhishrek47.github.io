// The Solution — Before vs After

Same output.
One command.

Everything in the manual workflow still happens — extract, strip, re-time, sync, burn, export. The script just does it without a human in the loop.

✗ Before — Manual (per language)
Export video from Storyline
Strip audio in Premiere / DaVinci

Manually re-time narration to video
Fix subtitle timing offsets

→
✓ After — Automated (all languages)

Output files named, formatted and ready to upload

How I Built This
✓ What's real in this demo
⌨FFmpeg commands shown are the actual commands from the production script — filter chains, audio normalisation flags and subtitle rendering options are verbatim
⏱Time figures are real — 36 hours manual vs under 8 minutes automated is the actual measured difference from the production workflow
🌐Language-specific log entries are accurate — DE narration really is ~7% longer than EN, AR really does require RTL rendering, ZH/JA require CJK font loading
⚡Parallel processing — the script processed all 8 languages simultaneously, not sequentially, which is why total time was under 8 minutes not 8 × n
~ What's simulated
💻Terminal execution — no server is running FFmpeg; the log output is a faithful reconstruction of actual script output, but rendered in JS
📁Output files — no video files are actually generated; the file paths and naming convention match the real output structure
→ Production architecture
localize.sh ──▶ Input validation ──▶ Language loop │ └──▶ FFmpeg // per language, parallel ├──▶ Extract video stream -vn flag ├──▶ Strip + replace audio -map 0:v -map 1:a ├──▶ Normalise levels loudnorm filter ├──▶ Burn SRT subtitles subtitles filter └──▶ Export final .mp4 -c:v copy │ └──▶ ./output/{LANG}/ ──▶ LMS upload ready
